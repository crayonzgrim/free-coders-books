# 신입 개발자 취업 전략

첫 개발자 직장을 구하는 것은 어렵습니다. 경력을 요구하는 공고, 수많은 경쟁자, 기술 면접의 압박... 하지만 **전략적으로 접근**하면 길이 보입니다.

## 현실 직시하기

### 채용 시장의 현실

- 신입에게 "경력 1-3년" 요구
- 지원자 > 채용 규모
- 학력/학벌 필터링 존재
- 기술 면접 난이도 상승

### 그래도 기회는 있다

- 개발자 수요는 여전히 높음
- 실력 있으면 학력 무관한 회사 많음
- 비전공자 출신 개발자 증가
- 스타트업 등 기회 다양

## 포트폴리오가 핵심

### 좋은 포트폴리오의 조건

**1. 완성된 프로젝트**

미완성 프로젝트 10개보다 **완성된 프로젝트 2-3개**가 낫습니다.

**2. 실제로 동작**

배포된 상태로 보여주세요:
- 웹: Vercel, Netlify
- 앱: TestFlight, 플레이스토어 테스트

**3. 코드 품질**

- 깔끔한 코드 구조
- README 문서화
- 커밋 히스토리

**4. 문제 해결 과정**

"왜 이 기술을 선택했는지", "어떤 문제를 어떻게 해결했는지" 설명할 수 있어야 합니다.

### 추천 프로젝트 유형

**Level 1: 기초**
- 투두 리스트 (CRUD 기본)
- 날씨 앱 (API 연동)
- 계산기 (UI/UX)

**Level 2: 중급**
- 블로그/게시판 (인증, 데이터베이스)
- 쇼핑몰 클론 (결제는 없어도 됨)
- 팀 프로젝트 (협업 경험)

**Level 3: 차별화**
- 실제 사용자가 있는 서비스
- 오픈소스 기여
- 기술 블로그 (꾸준한 글)

## 이력서 작성법

### 기술 스택

**나쁜 예:**
```
JavaScript, TypeScript, React, Vue, Angular, Svelte,
Node.js, Express, NestJS, Python, Django, Flask,
MySQL, PostgreSQL, MongoDB, Redis...
```

**좋은 예:**
```
주 기술: TypeScript, React, Next.js
사용 경험: Node.js, PostgreSQL
학습 중: Python
```

### 프로젝트 설명

**나쁜 예:**
> React와 Node.js를 사용해서 쇼핑몰을 만들었습니다.

**좋은 예:**
> **온라인 쇼핑몰 플랫폼** (2025.11 - 2025.12)
> - React, TypeScript, Zustand로 프론트엔드 구축
> - 상품 검색 최적화로 응답 시간 50% 단축
> - 이미지 lazy loading으로 초기 로딩 속도 개선
> - 배포: https://example.com | GitHub: ...

### 자기소개서

- **구체적인 경험** 중심
- **숫자**로 표현 (50% 개선, 3개 프로젝트, 6개월 학습)
- **왜 개발자인지** 명확히
- 회사별 **맞춤** 작성

## 기술 면접 준비

### 자주 나오는 주제

**프론트엔드**
- 브라우저 렌더링 과정
- React 동작 원리 (Virtual DOM, Reconciliation)
- 이벤트 루프, 비동기 처리
- 상태 관리
- 웹 성능 최적화

**백엔드**
- HTTP, REST API
- 데이터베이스 기초 (인덱스, 트랜잭션)
- 인증/인가 (JWT, 세션)
- 캐싱

**공통**
- 자료구조, 알고리즘 기초
- Git 사용법
- 내가 만든 프로젝트 설명

### 모르면 솔직히

"잘 모르겠습니다"가 "아는 척"보다 낫습니다. 대신:

> "정확히는 모르겠지만, 제가 알기로는 A인 것 같습니다. 맞나요?"

## 지원 전략

### 1. 타겟 설정

모든 회사에 지원하지 마세요. **10개를 잘 준비**하는 것이 **100개를 대충** 지원하는 것보다 낫습니다.

### 2. 회사 연구

- 어떤 기술 스택을 쓰는지
- 어떤 제품/서비스를 만드는지
- 최근 뉴스, 블로그

### 3. 지원 채널 다양화

- 공채
- 원티드, 로켓펀치 등 플랫폼
- 링크드인
- 개발자 커뮤니티
- 지인 추천

### 4. 거절에 담담하게

10곳 지원해서 1곳 면접 보면 잘한 겁니다. 거절은 **경험치**입니다.

## 첫 직장 선택

### 고려 사항

- **배울 수 있는 환경**: 시니어가 있는가? 코드 리뷰 문화가 있는가?
- **성장 가능성**: 회사가 성장하고 있는가?
- **워라밸**: 야근 문화, 휴가 사용
- **연봉**: 시장 평균 대비

### 첫 직장은 발판

첫 회사가 마지막 회사가 아닙니다. 1-2년 경력을 쌓으면 이직 시장에서 가치가 올라갑니다. 완벽한 첫 직장을 찾으려 하지 말고, **성장할 수 있는 곳**을 찾으세요.

## 마무리

취업 준비는 마라톤입니다. 당장 결과가 안 나와도 **꾸준히 준비하고 지원**하세요.

- 좋은 포트폴리오 만들기
- 기술 면접 준비하기
- 전략적으로 지원하기
- 거절에 흔들리지 않기

여러분의 첫 개발자 커리어를 응원합니다!
